Ingeniería de Software: Conceptos Fundamentales

Una infografía completa sobre los principios, procesos y prácticas esenciales en el desarrollo de software

¿Qué es la Ingeniería de Software?

Disciplina de la ingeniería que abarca todos los aspectos de la producción de software, desde la especificación hasta el mantenimiento.

Importancia: La sociedad depende cada vez más de sistemas de software avanzados. Es más económico aplicar métodos de ingeniería para asegurar confiabilidad y mantenimiento.

Ciclo de Vida del Software

  • Especificación: Definir qué debe hacer el sistema y sus restricciones
  • Desarrollo: Convertir especificaciones en un sistema ejecutable
  • Validación: Verificar que el sistema cumple con su especificación
  • Evolución: Modificar el software para requisitos cambiantes

Construcción de Software

Creación detallada del software mediante codificación, verificación, pruebas y depuración.

  • Transforma el diseño en código funcional
  • Varía según el modelo de ciclo de vida
  • Requiere atención al detalle y colaboración
  • Busca código eficiente, legible y escalable

Gestión de Cambios

  • Gestión de versiones: Seguimiento de versiones con control de versiones
  • Armado del sistema: Ensamblar componentes y compilar
  • Gestión de cambios: Evaluar costo e impacto de cambios
  • Gestión de liberación: Preparar software para distribución

Requisitos

Funcionales: Servicios o funcionalidades que el sistema debe proporcionar

No funcionales: Restricciones sobre los servicios o propiedades del sistema

Su claridad es crucial para el éxito del proyecto. Los no funcionales suelen ser más críticos.

Patrones de Diseño

  • Creacionales: Facilitan la creación de objetos (Abstract Factory, Singleton)
  • Estructurales: Definen relaciones entre clases y objetos (Adapter)
  • Comportamiento: Gestionan interacciones entre objetos (Observer, Decorator)

Son soluciones probadas para problemas típicos en desarrollo orientado a objetos.

Diseño de Software

  • Diseño del Sistema: Transforma requisitos de negocio en requisitos del sistema
  • Diseño Arquitectónico: Enfocado en la estructura de alto nivel del software
  • Diseño Detallado: Especifica cada componente con detalle para su construcción

Metodologías

Orientadas a Planes: Planificación detallada por adelantado (ej. Modelo en Cascada)

Ágiles: Planificación incremental y adaptativa (ej. Scrum)

La mayoría de proyectos combinan elementos de ambos enfoques según sus necesidades.
